Envision API Reference

Envision Company returns API

API Endpoint
envision-api.ap-southeast-2.elasticbeanstalk.com/api
Schemes: http
Version: 1.0.1

Paths

Public

Get company returns information

GET /v1.0

Get information about a particular companies returns.

Example usages:

Curl: curl -X GET "http://envision-api.ap-southeast-2.elasticbeanstalk.com/api/v1.0?instrument_id=ABP.AX&date_of_interest=2012-12-10&list_of_var=Return&lower_window=3&upper_window=5"

Python3: import requests r = requests.get('http://envision-api.ap-southeast-2.elasticbeanstalk.com/api/v1.0', params={ 'instrument_id': 'ABP.AX', 'date_of_interest': '2012-12-10', 'lower_window': 3, 'upper_window': 5, 'list_of_var' : 'Return' }) data = r.json()

JQuery: $.getJSON(`http://envision-api.ap-southeast-2.elasticbeanstalk.com/api/v1.0?instrument_id=${instrumentID}&date_of_interest=${dateOfInterest}&lower_window=${lowerWindow}&upper_window=${upperWindow}`, (data) => { let dataString = JSON.stringify(data, null, 3); })

instrument_id
in query
string

The tickers of financial instruments, in Reuters Instrument Code (e.g. CBA.AX) or natural ticker format (e.g. ASX:CBA). Comma seperated.

date_of_interest
in query
string (YYYY-MM-DD)

The desired date of interest.

list_of_var
in query
string

Which outputs should be included. Comma seperated string. Values may be: Returns, Return_pct, CM_Return, CM_Return_pct, AV_Return, AV_Return_pct, Daily_Spread, Volume, Volume_pct. See ouput description fo rthe meaning of these.

lower_window
in query
integer x ≥ 0

The number of days before the date of interest to include in the query.

upper_window
in query
integer x ≥ 0

The number of days after the date of interest to include in the query.

200 OK

Company returns data and metadata.

Date: The day that this object refers to.

AV_Return: Average returns from start day to this day. Only in response if it was requested through the list_of_var parameter.

AV_Return_pct: Average return percentage. Only in response if it was requested through the list_of_var parameter.

CM_Return: Cumulative returns from the start day to this day. Only in response if it was requested through the list_of_var parameter.

CM_Return_pct: Cumulative return percentage. Only in response if it was requested through the list_of_var parameter.

Daily_Spread: The difference between the highest and lowest trade price for that day. Only in response if it was requested through the list_of_var parameter.

Relative_Date: The number of days this day is from the date of interest. Only in response if it was requested through the list_of_var parameter.

Return: The return for that day. Only in response if it was requested through the list_of_var parameter. Only in response if it was requested through the list_of_var parameter.

Return_pct: Returns as a percentage of the previous days closing price. Only in response if it was requested through the list_of_var parameter.

Volume: Amount of the instrument traded on that day. Only in response if it was requested through the list_of_var parameter.

Volume_pct: Proportion of volume traded on that day compared to the sum of the sum of volume throughout the trading window. Only in response if it was requested through the list_of_var parameter.

elapsed_time: Time for the query to run.

end_time: End time of the query.

module: The name of the envision module.

parameters: The parameters of the query as envision understood them.

start_time: Start time of query.

success: Was the api call to envision successful.

team: The name of the team that made this api.

error_messages: An array of error messages. Only in response if there were errors.
